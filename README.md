# 考研备考助手系统交付说明

这是给甲方直接使用的交付包。

## 一、先做什么

请先安装并启动 **Docker Desktop**。

如果 Docker Desktop 没有启动，本系统无法运行。

---

## 二、如何启动系统

### Windows

在项目目录中，直接双击：

- `start-all.bat`

系统会自动启动：

- 数据库
- 后端服务
- 前端页面

### 第一次启动说明

第一次启动时，Docker 需要：

- 拉取镜像
- 构建前后端容器
- 初始化数据库

因此第一次启动会比以后慢一些，通常需要等待几分钟。

---

## 三、如何停止系统

### Windows

直接双击：

- `stop-all.bat`

它会停止当前系统容器。

---

## 四、如何重置数据库

如果需要恢复到演示初始数据，请双击：

- `reset-db.bat`

它会执行：

1. 删除当前数据库数据卷
2. 重新构建前后端镜像
3. 重新初始化数据库和演示数据

适用场景：

- 首次安装异常
- 演示数据被改乱了
- 需要恢复到初始状态重新演示

---

## 五、访问地址

启动完成后，可通过以下地址访问：

- 前端页面：`http://localhost:8081`
- 后端接口：`http://localhost:18080`
- 数据库端口：`localhost:3307`

一般只需要打开前端页面即可。

## 七、常见问题

### 1）双击 bat 没反应或提示 Docker 不可用

请确认：

- 已安装 Docker Desktop
- Docker Desktop 已经启动完成
- 任务栏中 Docker 图标状态正常

---

### 2）第一次启动很慢

这是正常现象。

第一次启动需要下载基础镜像并构建项目，请耐心等待。

后续再次启动会快很多。

---

### 3）端口被占用

本系统默认使用：

- `8081`：前端
- `18080`：后端
- `3307`：数据库

如果本机已有其他程序占用了这些端口，请先关闭冲突程序，或者联系技术人员调整 `docker-compose.yml` 里的端口映射。

---

### 4）页面打不开，但容器已经启动

请先等待 1~3 分钟再刷新页面。

因为数据库、后端、前端是按顺序启动的，首次构建时前端页面可访问前，后端和数据库需要先准备完成。

---

### 5）演示数据不对了

请直接双击：

- `reset-db.bat`

即可恢复到初始演示数据。

---
